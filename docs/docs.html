Nirik setup and self-hosting guide

Run Nirik on your own infrastructure to automate AI code reviews for GitHub pull requests and GitLab merge requests.

Quick start

  1. Clone the repository and create .env from .env.example.
  2. Add one AI provider key and provider tokens (GitHub and/or GitLab).
  3. Set a strong METRICS_TOKEN for /metrics access.
  4. Run with Docker Compose (docker compose up -d) or Node + Redis.

Required environment variables

  • GEMINI_API_KEY or OPENAI_API_KEY
  • GITHUB_TOKEN for GitHub PR reviews
  • GITLAB_TOKEN for GitLab MR reviews
  • METRICS_TOKEN for GET /metrics protection
  • REDIS_URL for queue + worker

Recommended: BASE_URL, GITHUB_WEBHOOK_SECRET, and GITLAB_WEBHOOK_TOKEN.

Webhook endpoint

Use this endpoint in GitHub and GitLab webhook settings:

POST /api/v1/webhooks/review-pr

Use pull/merge request events. Nirik processes open/update flows and posts review comments back on the same PR/MR.

Metrics endpoint security

/metrics is token-protected. Send:

Authorization: Bearer <METRICS_TOKEN>

Example:

curl -H "Authorization: Bearer your_token_here" https://your-domain.com/metrics

Production deployment notes

  • Run behind HTTPS (Nginx, Caddy, Traefik, or Cloudflare Tunnel).
  • Use long random values for webhook secrets and METRICS_TOKEN.
  • Monitor logs and queue health.
  • Pin BASE_URL to your external domain.
